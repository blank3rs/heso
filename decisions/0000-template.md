## Context

What is the problem we're solving? What constraints exist? What's the state of the world that makes this decision necessary?

Be specific. Quote numbers, file paths, library versions. A reader six months from now should be able to understand *why this came up* without context from the conversation that produced it.

## Decision

What are we doing? State it directly, in the active voice.

## Alternatives considered

For each meaningful alternative, name it and explain why it was rejected. Bias toward including alternatives that look obviously right at first glance — those are exactly the ones future readers will second-guess.

- **Option A:** ... — rejected because ...
- **Option B:** ... — rejected because ...

## Consequences

What does this decision enable? What does it cost? What does it foreclose?

Include both positive and negative consequences honestly. If you can't think of any negative consequences, you haven't thought hard enough.
