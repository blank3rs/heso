# Proposals (RFCs)

Design proposals for changes significant enough to need discussion before code. Promoted to an ADR (in `decisions/`) once decided, or marked withdrawn / declined.

## Index

Empty. Add proposals here using [`0000-template.md`](0000-template.md) as a starting point.

## When to write a proposal vs. an ADR

- **Proposal (here):** "Here's an idea, here's how it'd work, what do people think?" — pre-decision.
- **ADR (`decisions/`):** "Here's what we decided and why" — post-decision.

A proposal usually becomes an ADR after a brief discussion. Some proposals are withdrawn or rejected; those stay in this folder for historical record with status changed.

## Status values

- **Draft** — being written
- **Open** — awaiting feedback
- **Accepted** — moving to an ADR (link the ADR)
- **Rejected** — won't do, reasons documented
- **Withdrawn** — proposer pulled it
